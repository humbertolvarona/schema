Master every aspect of Mermaid diagrams with this comprehensive reference covering all diagram types, advanced features, real-world applications, and professional best practices.

1. Mermaid Fundamentals

1.1 Core Concepts

Mermaid transforms text definitions into diagrams using a simple but powerful syntax:

Basic Structure

%% Comments start with %%
diagramType Direction
    element1 --> element2
    subgraph Group
        element3
        element4
    end

Text Definition

graph TD
    A[Start] --> B{Decision}
    B -->|Yes| C[Action]
    B -->|No| D[Alternative]

Rendered Diagram

[Flowchart visualization]

Key Advantages

  • Version Control Friendly: Diagrams as code that can be diffed and merged
  • Dynamic Rendering: Diagrams update automatically when source changes
  • Wide Compatibility: Works in GitHub, GitLab, VS Code, Obsidian, and more
  • Extensive Customization: Full control over styling and layout

3. Sequence Diagrams: Professional Patterns

3.1 Core Syntax Elements

Participants and Messages

sequenceDiagram
    participant Client
    participant Server
    Client->>Server: Request
    Server-->>Client: Response
Element Syntax Description
Participant participant A Declares a diagram actor
Actor actor User Human participant (stick figure)
Sync Message A->>B: Message Solid line with arrowhead
Async Message A->)B: Message Open arrowhead
Return A-->>B: Response Dashed line
Self Message A->>A: Note Loopback message
Note Note right of A: Text Annotation on participant

3.2 Control Structures

Alternatives (if/else)

sequenceDiagram
    User->>System: Login
    alt Valid credentials
        System-->>User: Success
    else Invalid
        System-->>User: Failure
    end

Loops

sequenceDiagram
    loop Every minute
        Client->>Server: Heartbeat
        Server-->>Client: Ack
    end

Parallel Actions

sequenceDiagram
    par Parallel
        A->>B: Message 1
    and
        A->>C: Message 2
    end

Critical Region

sequenceDiagram
    critical Transaction
        A->>B: Begin
        B->>C: Commit
    option Timeout
        A->>B: Rollback
    end

Advanced Sequence Techniques

Activations
sequenceDiagram
    A->>+B: Request
    B->>+C: Query
    C-->>-B: Data
    B-->>-A: Response

Use + to activate and - to deactivate a participant's lifeline

Background Highlighting
sequenceDiagram
    rect rgba(0, 255, 0, 0.1)
        A->>B: Important Step
        B->>C: Verification
    end

Highlight sections with colored backgrounds

5. State Diagrams: Complex Systems

5.1 Basic State Machine

stateDiagram-v2
    [*] --> Idle
    Idle --> Processing: start
    Processing --> Success: complete
    Processing --> Error: failure
    Error --> Processing: retry
    Success --> [*]

5.2 Advanced State Features

Composite States

stateDiagram-v2
    [*] --> NotLoggedIn
    state Login {
        [*] --> Entering
        Entering --> Validating
        Validating --> Authenticated
    }
    NotLoggedIn --> Login
    Authenticated --> [*]

Concurrent States

stateDiagram-v2
    [*] --> Active
    state Active {
        [*] --> Downloading
        [*] --> Uploading
        Downloading --> Completed
        Uploading --> Completed
    }

State Diagram Techniques

History States
stateDiagram-v2
    [*] --> A
    A --> B
    B --> C
    C --> H: Remember
    state H {
        [*] --> A
    }
Notes and Annotations
stateDiagram-v2
    [*] --> Idle
    Idle --> Processing
    note right of Processing: This may take time
    Processing --> [*]

10. Professional Best Practices

10.1 Diagram Organization

  • Use meaningful, consistent naming conventions
  • Limit diagrams to 15-20 elements for readability
  • Break complex diagrams into smaller, focused ones
  • Use subgraphs and containers to group related elements
  • Add descriptive titles and legends

10.2 Version Control

  • Store Mermaid source files in Git (not just rendered images)
  • Use meaningful commit messages for diagram changes
  • Consider generating diagrams during build process
  • Maintain a diagram catalog or index

Advanced Workflows

Automated Documentation
# Generate diagrams from code annotations
/**
 * @mermaid
 * sequenceDiagram
 *   Client->>Server: API Call
 */
function apiCall() { ... }
CI/CD Integration
# Sample CI pipeline
steps:
  - name: Generate diagrams
    run: |
      npm install -g @mermaid-js/mermaid-cli
      mmdc -i docs/*.mmd -o static/images/

11. Troubleshooting and Debugging

11.1 Common Issues

Issue Solution
Blank diagram Check Mermaid is loaded and initialized
Syntax errors Validate in Mermaid Live Editor
Incorrect layout Verify direction (TD, LR etc.) and subgraphs
Missing features Check Mermaid version requirements
Rendering issues Try simpler diagram to isolate problem

11.2 Debugging Tools

  • Mermaid Live Editor for testing
  • Browser developer tools for integration issues
  • Mermaid's logLevel setting for verbose output
  • Simplified test cases to isolate problems

Advanced Debugging

Error Logging
mermaid.initialize({
    logLevel: 1, // 1-5 (verbose)
    securityLevel: 'loose'
});
Custom Error Handling
try {
    mermaid.parse(text);
} catch (err) {
    console.error('Mermaid error:', err);
}
